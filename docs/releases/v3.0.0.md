# Release Notes — v3.0.0

**Release date**: 2026-04-22  
**Branch**: `ship/audit-fixes-and-cleanup`  
**Status**: Development / Pre-release

## Breaking Changes from v2.x

### 1. Python Scanner Removed

The Python scanner at `apps/scanner/` has been permanently deleted. The **Go scanner** at `apps/goScanner/` is the canonical, sole scanner.

- **Old**: Python scanner on port `:5000`, Flask HTTP server
- **New**: Go scanner on port `:8001`, internal Docker network only

Update any CI scripts, Docker Compose files, or deployment configs that reference:
- `apps/scanner/`
- Port `5000` for the scanner
- `python hawk_scanner/main.py`
- `pip install -r apps/scanner/requirements.txt`

Replace with the Go scanner. See [docs/SCANNER_REFERENCE.md](../SCANNER_REFERENCE.md).

### 2. Health Endpoints Renamed

| Old | New | Notes |
|-----|-----|-------|
| `GET /health` | `GET /livez` | **Liveness** — always `200` while process runs |
| `GET /health` | `GET /readyz` | **Readiness** — `503` if PostgreSQL or Neo4j unreachable |

`GET /health` is retained as a back-compat alias for `/readyz`. Update Kubernetes probes and monitoring to use `/livez` and `/readyz` respectively.

### 3. Scanner Port Changed

The Go scanner does not publish port `8001` to the host. It is reachable only on the internal Docker bridge (`arc-platform-internal`). External tooling that previously called the scanner directly must route through the backend API instead.

### 4. Celery Removed

Celery and the Python task queue have been removed along with the Python scanner. Async job execution is handled entirely by **Temporal**.

### 5. `SCANNER_URL` Points to Go Scanner

The backend env var `SCANNER_URL` now resolves to `http://go-scanner:8001`. If you override this in your deployment, update accordingly.

## New Features

### Go Scanner with 36+ Connectors

The Go scanner (`apps/goScanner/`) replaces the Python scanner with:
- 36+ production connectors across databases, cloud storage, SaaS, and file formats
- Per-field streaming pipeline (channel-based) — lower memory than batch processing
- Self-registration pattern — connectors register themselves; no central registry file to update
- Configurable `SCAN_TIMEOUT_SECONDS` wall-clock cap per scan

### Per-Finding Risk Scores (migration `000044`)

Each finding now carries a `risk_score` integer (0–100) computed from confidence, PII type sensitivity, and asset exposure. The `findings` table has a new `risk_score` column. Existing rows default to `0` and are back-filled by the enrichment service on the next scan.

### Streaming Ingestion

The scanner streams results to the backend in configurable batches rather than one large payload at end-of-scan. This reduces peak memory usage and enables real-time progress reporting.

### Neo4j + Temporal TLS Enforcement

In release mode (`GIN_MODE=release`):
- Neo4j connections must use `bolt+s://` or `bolt+ssc://` URI schemes
- Temporal connections verify server certificates

### Scanner Auth Token (`SCANNER_SERVICE_TOKEN`)

A shared secret now authenticates backend ↔ scanner communication. The token is validated with constant-time comparison in both directions. In release mode, the token must be at least 32 characters and must not equal the placeholder value.

### Remediation Preview Gating

`POST /api/v1/remediation/execute` is disabled by default (`REMEDIATION_ENABLED=false`). Set `REMEDIATION_ENABLED=true` only after verifying that connector-side mutation is implemented and tested. The preview endpoint (`POST /api/v1/remediation/preview`) is always available.

### Mobile Navigation

The frontend navigation is now responsive and functional on mobile viewports (hamburger menu, slide-out drawer).

### False-Positive Learning (migration `000045`)

The `fplearning` module records analyst feedback on findings (true positive / false positive). This data is persisted in the `fp_learning` table and is available for future classifier tuning.

## Migration Notes

### Run All Versioned Migrations

After upgrading from v2.x, run all migrations up to `000045`:

```bash
migrate \
  -path apps/backend/migrations_versioned \
  -database "postgres://..." \
  up
```

New migrations introduced in v3.0.0:
- `000043` — remediation_previews table
- `000044` — finding_risk_score column on findings
- `000045` — fp_learning table

### Update Scanner Port References

Search your configs for port `5000` and update to `8001`. The Go scanner does not expose `8001` to the host — all scanner access routes through the backend.

```bash
grep -r "5000\|scanner.*flask\|apps/scanner" infra/ helm/ .github/ --include="*.yaml" --include="*.yml"
```

### Kubernetes Probes

Update liveness and readiness probes:

```yaml
livenessProbe:
  httpGet:
    path: /livez
    port: 8080

readinessProbe:
  httpGet:
    path: /readyz
    port: 8080
```
