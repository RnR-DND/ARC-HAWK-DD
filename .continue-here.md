## ACTIVE BUG: Scanner Returns 0 Findings

### Symptom
When adding a PostgreSQL data source and running a scan, it completes in ~2 seconds with 0 findings. Scanner logs show:
```
Using 1 runtime connection configs from backend
Executing: hawk_scanner all --json /tmp/scan_output_xxx.json --connection /tmp/connection_xxx.yml
Raw results keys: []
Scan xxx completed with zero findings — nothing to ingest
```

### Diagnosis So Far
- hawk_scanner CLI runs successfully (no errors, exit code 0)
- But output JSON is empty `{}`
- The connection config IS being passed from backend (log confirms "1 runtime connection config")
- The scanner writes a temp YAML file at `/tmp/connection_xxx.yml` and passes it to hawk_scanner

### Likely Root Causes (investigate in order)
1. **Field name mismatch** — The frontend stores connection config with field names that may not match hawk_scanner's expected format. hawk_scanner expects: `host`, `port`, `user`, `password`, `database`. The frontend may send `hostname`, `username`, etc. Check the ScanConfigModal and AddSource components to see what field names the frontend uses.
2. **Profile name mismatch** — `resolveConnectionConfigs()` in `scan_trigger_handler.go:253` matches `conn.ProfileName` against `req.Sources[]`. If the scan request sends different source names than what's stored as profile_name in the connections table, the config won't resolve. Add logging to see what's being matched.
3. **YAML structure issue** — The temp YAML written by scanner_api.py may not match hawk_scanner's expected structure. Need to add debug logging to print the actual YAML content before hawk_scanner runs.

### How to Debug
1. Add debug logging in `scanner_api.py` around line 152 to print the YAML content:
   ```python
   with open(connection_config_path, 'w') as f:
       yaml.dump(connection_data, f)
   # DEBUG: print actual YAML
   with open(connection_config_path, 'r') as f:
       logger.info(f"Connection YAML content:\n{f.read()}")
   ```
2. Check what the frontend sends: look at `ScanConfigModal` component to see the `sources` array format
3. Check what's stored: `docker exec arc-platform-db psql -U postgres -d arc_platform -c "SELECT profile_name, source_type FROM connections;"`
4. Compare field names: check frontend AddSource form field names vs hawk_scanner expected format (see `config/connection.yml.sample` lines 81-89)
